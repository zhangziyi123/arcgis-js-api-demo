CreateDrive-TimeAreas

Create Drive-Time Areas shows what areas can be reached within a specified drive time or drive distance. It measures out from one or more points, along roads, to create a layer that can help you answer questions such as:

  • Where can I go from here within a 30-minute drive?
  • Where can I go from here within a 30-minute drive at 6:00 p.m. during rush hour?
  • What areas of town can the fire department reach within five minutes? How would fire-response coverage improve by building a new fire station here?
  • What market areas does my business cover? What areas are within 10 miles of my stores?

Tip: 

To create areas by measuring straight-line distance—that is, as the crow flies—rather than by measuring along roads, use Create Buffers.

Choose points

Select an input point layer from the web map. The drive-time areas will be generated around these points.

Measure

Choose whether to calculate drive-time or drive-distance areas.

  • Driving Time—The output areas show how far you can drive within the drive time you specify. The tool calculates the areas assuming average speeds along streets or, if you specify a start time, the estimated speeds given traffic conditions for that time and day.
  • Driving Distance—The distance you specify is measured along roads.

In both cases, one-way streets, illegal turns, and so on, are considered in generating the areas.

Drive time or distance

Type a drive time or drive distance and specify the units.

To create multiple areas of different sizes around each point, type multiple values. Separate the values with spaces. For instance, to find 5-, 10-, and 15-minute drive-time areas, type the following: 5 10 15.

Measurement Units

Choose a time or distance unit to help define the size of the areas to create.

If Measure is set to Driving Time, only time units are shown here, not distance units. To see distance units, set Measure to Driving Distance.

Use Traffic Conditions

Choose whether drive-time areas are created by using average speeds or speeds based on changing traffic conditions. By checking this option, you enable the parameters that allow you to specify a time and day of departure.

If you choose to use traffic in the analysis, the tool will automatically read the most accurate information available: historical, live, or predictive traffic. Historical traffic stores a record of average travel speeds at different times of the day for each day of a typical week. Live traffic uses near-current travel speeds obtained from sensors in cars and on roads. Predictive traffic uses a mix of live traffic, historical traffic, and other information to make forecasts of travel speeds.

Not all regions support historical, live, and predictive traffic. See availability.

Use Traffic Conditions is enabled when Measure is set to Driving Time and disabled otherwise.

Time of Departure

The time to depart from the input point locations.

Given a constant drive time, the size of a drive-time area can change substantially depending on the departure time and, consequently, traffic. The area tends to be smaller during peak traffic hours and larger in the middle of the night when fewer cars are on the roads.

When calculating drive-time areas, the tool accounts for how traffic and travel speeds change throughout the drive. For example, if you specify a 30-minute drive-time area with a departure time of 8:00 a.m., the travel speeds at the beginning of the trip (that is, near the point of departure) are calculated from 8 o'clock traffic. But as the trip continues and time elapses, traffic for different times of day are referenced. By the end of the 30-minute drive time, near the boundary of the drive-time area, the tool is referencing 8:30 a.m. traffic, not 8:00 a.m. traffic.

This parameter is enabled when Measure is set to Driving Time and Use Traffic Conditions is checked.

Day of Week

The day to which the Time of Departure parameter refers. You can choose a day of the week or Today.

Areas from different input points should

Choose how shared areas from different input points are handled. They can:

  • Overlap—Drive-time areas from different input points may cover the same road. This is useful when you need to know all the input points that can reach a particular location within the given drive time or distance.
  • Dissolve—All drive-time areas are merged into one, and no more than one area can cover a particular location. This is useful when all you need to know is that an area can be reached within a given time or distance, but you don't need to know which specific input points can reach the location.
  • Split—The drive-time areas from different input locations that would overlap each other are split in the middle, so they about one another instead. This is useful when you want to see which areas are within a given time or distance of the input points and, at the same time, determine which input point is closest to that area.

Result Layer Name

The result layer name will be generated by default with the name being based on the analysis type and the layer name of the input features. To change the result layer name, click inside the dialog box to edit.

If a layer already exists with the same name, you will be prompted to confirm if you want to overwrite the original layer.
