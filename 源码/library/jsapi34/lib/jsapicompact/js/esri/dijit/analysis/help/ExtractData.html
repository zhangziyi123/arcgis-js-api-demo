ExtractData

Using Extract Data you can select and download data for a specified area of interest. Layers that you select will be added to a zip file or layer package.

Layers to Extract

The layers in your table of contents from which you can extract features. Layers that are checked are extracted and included in a resulting zip file or layer package.

Study area

The area that defines which features will be included in the output zip file or layer package. You can choose from the following:

  • The current visible extent of your map ( Same as display)
  • One of the layers in your table of contents (that contains area features)
  • Draw one or more areas to use
All features completely within the extent will be extracted. For features that cross or intersect the extent boundary but are not completely within the extent, you have two options:

  • Select features — Entire features that cross the boundary of the study area will be extracted.
  • Clip features — Only those portions of features that are within the study area will be extracted.

Output data type

Format of the data that will be extracted and downloaded. Layer packages will always include file geodatabases.

Output file name

The default result layer name is based on the chosen Output data format. Choosing either File geodatabase or Shapefile creates a .zip file. If Layer package is chosen, an .lpk file will be created.

If the item already exists you will be asked to confirm if you want to overwrite it.
